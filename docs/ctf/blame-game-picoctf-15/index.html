Blame Game (picoCTF 15)

Challenge Author: Jeffery John

Difficulty: Easy

Description

Someone’s commits seem to be preventing the program from working. Who is it?

Process / Notes

  1. wget and unzip the files
  2. cat the message.py
  3. Looks like a half-done print statement
  4. git log to see the log
  5. It looks like one author is committing “important business work” an uncountable amount of times, at the same time
  6. git branch -a shows only the master branch is there
  7. cat a bunch of things - no clues
  8. cat HEAD gave a reference location, trying to cat refs/heads/master
  9. Got a git code, seems to point to the ee09 branch
  10. git checkout to that branch, seems the same. That might be the most recent one?
  11. Going to pipe the git log into a text file and go to the bottom with Neo Vim
  12. The author for the second commit looks to be the flag!
  13. I’m curious what these first two commits looked like
  14. It’s just the full code and the edit where someone removed the last parenthesis

11 minutes 43 seconds to complete

Hints

  1. In collaborative projects, many users can make changes
  2. Read the chapter on Git from the picoPrimer here
  3. You can use the python3 .py to try running the code, though you won’t nee to for this challenge

Core Lessons

  1. Understanding how to use git log to see the history of changes and commits in a git project
  2. Understanding how to navigate a large git log to find where the breaks occured and who made those changes
